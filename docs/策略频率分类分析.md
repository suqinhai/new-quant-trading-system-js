### 2.2 SMA策略详解

**文件**: `src/strategies/SMAStrategy.js`

**策略原理**:
- 使用短期均线和长期均线的交叉作为买卖信号
- 金叉 (短期上穿长期) = 买入信号
- 死叉 (短期下穿长期) = 卖出信号

**默认参数**:
```javascript
{
  shortPeriod: 10,      // 短期均线周期
  longPeriod: 30,       // 长期均线周期
  symbol: 'BTC/USDT',   // 交易对
  positionPercent: 95   // 仓位比例
}
```

**交易频率分析**:
- 日线级别: 月均 1-3 次信号
- 4H级别: 月均 5-15 次信号
- 1H级别: 月均 20-50 次信号 (不推荐,假信号多)

**适用场景**:
- 趋势明显的市场
- 长期投资组合
- 大资金低频操作

### 2.3 轮动策略详解

**文件**: `src/strategies/RotationStrategy.js`

**策略原理**:
1. 计算所有币种的综合强弱分数
2. 持有最强的 Top N 资产 (做多)
3. 做空最弱的 Bottom N 资产 (可选)
4. 按周期或触发条件进行轮动调仓

**默认参数**:
```javascript
{
  // 轮动配置
  rebalancePeriod: 7 * 24 * 60 * 60 * 1000,  // 每周轮动
  lookbackPeriod: 14,           // 14日回看周期

  // 选股配置
  topN: 5,                      // 持有最强5个
  bottomN: 3,                   // 做空最弱3个

  // 强弱指标
  strengthMetric: 'composite',  // 复合指标

  // 缓冲区 (防止频繁换仓)
  useBufferZone: true,
  bufferZoneSize: 2,
  minHoldingPeriod: 24 * 60 * 60 * 1000,  // 最少持仓1天

  // 监控币种
  symbols: [
    'BTC/USDT', 'ETH/USDT', 'BNB/USDT', 'SOL/USDT', 'XRP/USDT',
    'ADA/USDT', 'AVAX/USDT', 'DOGE/USDT', 'DOT/USDT', 'MATIC/USDT',
    // ... 更多币种
  ]
}
```

**强弱分数计算方式**:
```javascript
// 复合强弱分数 = 加权综合
compositeScore = (
  momentum * 0.25 +      // 动量 25%
  returns * 0.20 +       // 收益率 20%
  sharpe * 0.20 +        // 夏普比率 20%
  relativeStrength * 0.20 + // 相对强弱 20%
  trendStrength * 0.15   // 趋势强度 15%
) * rsiAdjust;           // RSI调整因子
```

**交易频率**: 每周 1 次定期轮动 + 异常情况触发轮动

### 3.2 RSI策略详解

**文件**: `src/strategies/RSIStrategy.js`

**策略原理**:
- RSI < 超卖阈值 (30) = 买入信号
- RSI > 超买阈值 (70) = 卖出信号

**默认参数**:
```javascript
{
  period: 14,           // RSI计算周期
  overbought: 70,       // 超买阈值
  oversold: 30,         // 超卖阈值
  symbol: 'BTC/USDT',
  positionPercent: 95
}
```

**参数优化建议**:
| 市场状态 | overbought | oversold | 说明 |
|---------|-----------|----------|------|
| 强趋势 | 75-80 | 20-25 | 更保守,避免逆势 |
| 震荡市 | 65-70 | 30-35 | 更积极,捕捉区间 |
| 高波动 | 80 | 20 | 极端值才交易 |

### 3.3 自适应策略详解

**文件**: `src/strategies/AdaptiveStrategy.js`

**策略原理**:
- 根据市场波动率动态调整指标参数
- 高波动 → 缩短周期,快速反应
- 低波动 → 延长周期,减少噪音

**自适应参数调整逻辑**:
```javascript
// SMA自适应
{
  smaBaseFast: 10,              // 基准快速周期
  smaBaseSlow: 30,              // 基准慢速周期
  smaPeriodAdjustRange: 0.5,    // 调整范围 ±50%
  smaVolLowThreshold: 25,       // 低波动百分位
  smaVolHighThreshold: 75,      // 高波动百分位
}

// RSI自适应阈值
{
  // 趋势市 (大幅波动)
  rsiTrendingOversold: 25,
  rsiTrendingOverbought: 75,

  // 震荡市 (窄幅波动)
  rsiRangingOversold: 35,
  rsiRangingOverbought: 65,
}

// 布林带自适应
{
  bbBaseStdDev: 2.0,    // 基准标准差
  bbMinStdDev: 1.5,     // 低波动时
  bbMaxStdDev: 3.0,     // 高波动时
}
```

### 4.2 网格策略详解

**文件**: `src/strategies/GridStrategy.js`

**策略原理**:
- 在价格区间内设置多个网格线
- 价格下穿网格线 → 买入
- 价格上穿网格线 → 卖出
- 实现自动低买高卖

**默认参数**:
```javascript
{
  upperPrice: 50000,      // 网格上限
  lowerPrice: 30000,      // 网格下限
  gridCount: 10,          // 网格数量
  amountPerGrid: 100,     // 每格投资额(USDT)
  symbol: 'BTC/USDT'
}
```

**网格计算**:
```
网格间距 = (50000 - 30000) / 10 = 2000 USDT

网格线价格:
  Grid 0: 30000 (最低)
  Grid 1: 32000
  Grid 2: 34000
  ...
  Grid 9: 48000
  Grid 10: 50000 (最高)
```

**交易频率**: 取决于价格波动幅度,震荡市可达日均 10-50 次

**适用场景**:
- 震荡行情
- 有明确支撑阻力的区间
- 小资金高频操作

### 4.3 订单流策略详解

**文件**: `src/strategies/OrderFlowStrategy.js`

**策略原理**:
基于实时成交数据分析市场微观结构:
1. **成交量突增** - 检测异常放量
2. **VWAP偏离** - 价格相对VWAP的偏离度
3. **大单/小单比例** - 机构 vs 散户行为
4. **主动买卖比例** - Taker Buy Ratio

**默认参数**:
```javascript
{
  // 成交量突增
  volumeMAPeriod: 20,
  volumeSpikeMultiplier: 2.0,   // 2倍均量视为突增

  // VWAP
  vwapPeriod: 20,
  vwapDeviationThreshold: 1.0,  // 偏离1%触发信号

  // 大单
  largeOrderMultiplier: 3.0,    // 3倍平均成交量为大单
  largeOrderRatioThreshold: 0.6, // 大单买入占比60%

  // Taker
  takerWindow: 10,
  takerBuyThreshold: 0.6,       // 60%主动买入
  takerSellThreshold: 0.4,      // 40%主动卖出

  // 信号组合
  minSignalsForEntry: 2,        // 至少2个指标确认

  // 止损止盈
  stopLossPercent: 1.5,
  takeProfitPercent: 3.0,
  useTrailingStop: true,
  trailingStopPercent: 1.0
}
```

**信号生成逻辑**:
```javascript
// 多头信号条件 (至少满足2个):
// 1. 放量上涨 (成交量 > 2x均量 且 阳线)
// 2. 价格高于VWAP 1%以上
// 3. 大单买入占比 > 60%
// 4. 主动买入占比 > 60%

// 入场: bullishCount >= 2 && bullishStrength > bearishStrength
```

### 4.4 资金费率套利策略详解

**文件**: `src/strategies/FundingArbStrategy.js`

**策略原理**:
- 监控多个交易所的永续合约资金费率
- 当费率差异显著时,在高费率交易所做空 (收取费用),在低费率交易所做多 (支付较少费用)
- 赚取费率差异的收益

**支持的交易所**:
- Binance
- Bybit
- OKX
- Gate
- Deribit
- Bitget

**默认参数**:
```javascript
{
  // 交易对
  symbols: ['BTC/USDT', 'ETH/USDT'],

  // 套利阈值
  minAnnualizedSpread: 0.15,    // 15%年化利差开仓
  closeSpreadThreshold: 0.05,   // 5%年化利差平仓
  emergencyCloseThreshold: -0.10, // -10%紧急平仓

  // 仓位配置
  maxPositionSize: 10000,       // 单个套利最大10000 USDT
  minPositionSize: 100,
  positionRatio: 0.25,          // 单次开仓25%
  totalMaxPosition: 50000,

  // 杠杆
  leverage: 5,
  maxLeverage: 10,

  // 监控间隔
  fundingRefreshInterval: 30000,   // 30秒刷新费率
  positionRefreshInterval: 10000,  // 10秒检查持仓
  rebalanceInterval: 60000,        // 1分钟再平衡检查

  // 风控
  maxDailyLoss: 500,
  maxDrawdown: 0.10,
  imbalanceThreshold: 0.10,     // 10%仓位不平衡触发再平衡
}
```

**年化利差计算**:
```javascript
// 资金费率每8小时结算一次
// 每年结算次数 = 365 * 3 = 1095 次

annualizedSpread = (shortRate - longRate) * 1095

// 例: Binance费率 0.01%, OKX费率 0.005%
// 年化利差 = (0.01% - 0.005%) * 1095 = 5.475%
```

### 4.5 统计套利策略详解

**文件**: `src/strategies/StatisticalArbitrageStrategy.js`

**4种套利形式**:

| 类型 | 英文名 | 原理 |
|-----|-------|------|
| 协整交易 | Cointegration | 两个资产存在长期均衡关系 |
| 配对交易 | Pairs Trading | 价差的均值回归 |
| 跨所套利 | Cross-Exchange | 同资产不同交易所价差 |
| 期现基差 | Perpetual-Spot | 永续合约与现货的基差 |

**默认参数**:
```javascript
{
  // 策略类型
  arbType: 'pairs_trading',

  // 配对配置
  candidatePairs: [
    { assetA: 'BTC/USDT', assetB: 'ETH/USDT' },
    { assetA: 'ETH/USDT', assetB: 'BNB/USDT' },
    { assetA: 'SOL/USDT', assetB: 'AVAX/USDT' },
  ],
  maxActivePairs: 5,
  lookbackPeriod: 60,

  // Z-Score信号
  entryZScore: 2.0,         // Z-Score >= 2.0 开仓
  exitZScore: 0.5,          // Z-Score <= 0.5 平仓
  stopLossZScore: 4.0,      // Z-Score >= 4.0 止损
  maxHoldingPeriod: 7 * 24 * 60 * 60 * 1000, // 最多持仓7天

  // 跨交易所套利
  spreadEntryThreshold: 0.003,  // 0.3%价差开仓
  spreadExitThreshold: 0.001,   // 0.1%价差平仓
  tradingCost: 0.001,           // 0.1%手续费
  slippageEstimate: 0.0005,     // 0.05%滑点

  // 期现基差
  basisEntryThreshold: 0.15,    // 15%年化基差开仓
  basisExitThreshold: 0.05,     // 5%年化基差平仓
}
```

## 六、策略频率与风险特征

### 6.1 风险收益对比

| 指标 | 低频策略 | 中频策略 | 高频策略 |
|-----|---------|---------|---------|
| **胜率** | 40-55% | 50-60% | 55-70% |
| **盈亏比** | 2:1 ~ 5:1 | 1.5:1 ~ 3:1 | 0.5:1 ~ 1.5:1 |
| **最大回撤** | 15-30% | 10-20% | 5-15% |
| **回撤恢复** | 慢 (周级) | 中 (日级) | 快 (小时级) |
| **资金效率** | 低 | 中 | 高 |
| **手续费占比** | 低 (<5%) | 中 (5-15%) | 高 (15-40%) |

### 6.2 不同市场环境表现

| 市场环境 | 低频表现 | 中频表现 | 高频表现 |
|---------|---------|---------|---------|
| **强趋势** | 优秀 | 良好 | 一般 |
| **震荡市** | 较差 | 良好 | 优秀 |
| **高波动** | 一般 | 优秀 | 良好 |
| **低波动** | 较差 | 一般 | 优秀 (套利) |
| **黑天鹅** | 风险大 | 风险中 | 风险小 |

```

│   ├── AdaptiveStrategy: 15%
│   └── RiskDrivenStrategy: 15%
└── 高频策略: 10%
    └── GridStrategy: 10%
```

### 7.2 市场环境策略选择

| 市场状态 | 识别方法 | 推荐策略 |
|---------|---------|---------|
| **强上涨趋势** | MA多头排列, ADX>30 | SMA, MACD, Momentum |
| **强下跌趋势** | MA空头排列, ADX>30 | SMA(做空), Rotation(空头) |
| **横盘震荡** | ADX<20, 布林带收窄 | Grid, RSI, Bollinger |
| **高波动突破** | ATR突增, 布林带扩张 | ATRBreakout, Adaptive |
| **低波动期** | VIX低位, ATR缩小 | FundingArb, StatArb |
